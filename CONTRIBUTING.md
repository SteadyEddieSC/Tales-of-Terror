# Contributing

1. Link work to a GitHub issue.
2. Use a short-lived `feature/`, `fix/`, or `docs/` branch.
3. Keep changes small enough to review and test.
4. Update design/technical documentation and ADRs when behavior or architecture changes.
5. Never commit secrets, signing keys, engine caches, build outputs, or unlicensed assets.
6. Record third-party asset and dependency provenance.
7. Squash merge after required checks pass.

The repository is public, but the final noncommercial source-available license has not yet been selected. External contributions should not be accepted until contribution and licensing terms are explicit.
