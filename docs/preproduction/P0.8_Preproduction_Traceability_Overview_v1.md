P0.8 consolidates the seven merged preproduction packages into one discoverable and machine-validated body of work.

1. Which package governs each part of the preproduction work?

3. Which meanings and privacy boundaries may not drift between media?
4. What must Codex do first when local Windows access returns?

## Non-negotiable cross-media rules

### Stable seats persist

No medium may turn human departure, game control, takeover, return, defeat, Tidebound transformation, or Restless continuation into:

- character replacement;
- reset;
- respawn;
- automatic healing;
- inventory restoration;
- player elimination;
- an NPC conversion.

### Hidden state remains hidden

No visual cue, SFX, music stem, voice performance, caption, transcript, or accessibility announcement may expose:

- hidden factions;
- private objectives;
- private bargain terms;
- latent transformations;
- private inventory or routes;
- hidden Director targets;
- pending private choices;
- unrevealed ending carriers.

### Mechanical meaning outranks atmosphere

The same public event must retain one meaning across:

- authored design;
- visual presentation;
- SFX;
- music;
- voice;
- captions;
- transcript;
- assistive announcements.

Atmosphere may intensify an event. It may not change its legal action, certainty, timing, privacy, stable-seat continuity, or consequence.

### Human evidence remains required

Automation and schemas do not prove:

- fun;
- balance;
- living-room readability;
- controller comfort;
- social tension;
- audio quality;
- music fatigue;
- voice performance;
- translation quality;
- caption pacing;
- screen-reader behavior;
- accessibility compliance.

## Validation

Run:

```bash
python tools/validate_preproduction_package_traceability.py
python tools/test_validate_preproduction_package_traceability.py
```

The validator rejects:

- a missing P0.1–P0.7 package;
- incorrect release order, pull request identity, or squash lineage;
- nonexistent release summaries or controlling paths;
- implementation or production-asset approval;
- missing issues #7, #39, or #44;
- unknown audio, music, voice, or accessibility IDs;
- voice and accessibility coverage drift;
- signature concepts without visual, audio, or music coverage;
- concepts that no longer require human validation;
- implementation-ready status;
- a destructive or incomplete Codex handoff.
