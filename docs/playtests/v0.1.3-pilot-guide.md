# v0.1.3 First Pilot Facilitator Guide

This kit prepares one consent-based household in-room or remote-observed pilot. It does not claim that any human, physical-controller, display, phone, network, accessibility, duration, balance, or fun observation has occurred.

`Terror Turn` is the exact current Godot project-folder name and remains a provisional working title pending issue #7.

## 1. Verify the candidate before sharing or launching

Use only the platform artifact named in the `PILOT-READY HANDOFF`. Compare its GitHub artifact size and SHA-256 with that handoff. Extract the GitHub artifact, verify the inner ZIP with its adjacent `.zip.sha256`, and open the ZIP without warnings so its CRC is checked.

- Windows PowerShell: `Get-FileHash -Algorithm SHA256 <archive.zip>`
- Windows Command Prompt: `certutil -hashfile <archive.zip> SHA256`
- Linux: `sha256sum -c <archive.zip.sha256>`

After extracting the inner ZIP, open `build_manifest.json`. Confirm release `v0.1.3`, the exact candidate source SHA, intended platform, `x86_64`, and `internal_playtest_not_public_release`. In the game, X/H opens Help; Build & Support must show the same short source, target, and `INTERNAL PLAYTEST` classification. Stop if any value differs.

## 2. Pre-session setup

- Use a normal user-writable folder; do not run as administrator.
- Connect intended controllers before launch. Do not record controller names, serial numbers, device IDs, usernames, IP addresses, or exact household/network details.
- Windows: start `launch.cmd` or the adjacent `.exe`. Linux: start `./launch.sh` or the adjacent `.x86_64` file.
- Confirm the display is visible from the intended seating position without photographing participants or the room.
- Keep phones out of the primary route. The complete route must work with no phone. Optional companion observation is separate and is not required.
- Have `OBSERVATION_SHEET.md` and a fresh copy of `PILOT_SESSION_RECORD.json` available locally. Do not enter a repository path or private report content.

## 3. Opening script

Read this before play:

> This is an unfinished internal game pilot, not a test of you. Participation is voluntary. You may pause, skip a question, or stop at any time without giving a reason. We will record only de-identified gameplay observations and broad aggregate counts. We do not need names, contact details, exact ages, addresses, account names, photographs, audio/video, medical details, device identifiers, room codes, or network information. Please avoid saying private role information for the record. One short pilot cannot certify fun, balance, duration, accessibility, privacy, security, or production readiness.

For a youth participant, a household parent or guardian controls participation. Record only `youth_with_guardian`; never record an exact age, birth date, school, medical detail, or identity.

## 4. Facilitator sequence

1. Record only the verified artifact identity; create a non-identifying session code shaped like `PILOT-XXXXXXXX` and record only the calendar date.
2. Ask players to discover title -> stable-seat lobby -> confirmation -> briefing -> active tale -> ending with no phone. Avoid coaching until assistance is needed; count assistance without identifying a player.
3. Mark an observation only when actually seen. Use `participant_reported` only for a participant's voluntary statement; do not infer emotion, intent, consent, fun, accessibility, or severity.
4. Attempt local report export at ending. Record only SHA-256 hashes of reviewed JSON/Markdown files, never their contents or filenames.
5. If reasonable, observe return-to-title or rematch. Protected reset/reconnect may be deferred rather than disrupting the session.
6. Ask the bounded questionnaire. Participants may skip any question.
7. Review the copied record locally for prohibited data before returning it.

## 5. Interruption and recovery

- Pause immediately if anyone asks. Stopping is always permitted.
- For a disconnect, keep the stable seat reserved and use the normal reconnect guidance; record only the category and outcome.
- For a crash or launch failure, preserve the candidate identity and exact visible error only. Do not collect OS usernames, paths, dumps, screenshots containing identities, or device/network details.
- Do not restart with a different build. If code changes are needed, stop and request a new frozen candidate handoff.
- Do not call a short restart sequence long-session evidence.

## 6. Reports and evidence return

Reports remain local under:

- Windows: `%APPDATA%\Godot\app_userdata\Terror Turn\playtest_exports`
- Linux: `$XDG_DATA_HOME/godot/app_userdata/Terror Turn/playtest_exports`
- Linux fallback: `~/.local/share/godot/app_userdata/Terror Turn/playtest_exports`

Review local reports privately. Put hashes only in `PILOT_SESSION_RECORD.json`. Do not return raw reports. The evidence package must contain exactly one file named `PILOT_SESSION_RECORD.json`; unknown files, nested folders, links, oversized files, private paths, secrets, room codes, IPs, usernames, device IDs, tokens, and direct identifiers reject. Return the package through the private channel selected by the user/main chat; this repository provides no uploader, telemetry, cloud API, or network client.

Run offline intake only from a trusted repository checkout:

`python tools/pilot_evidence.py normalize --package-dir <reviewed-folder> --expected-identity <candidate-identity.json> --output <normalized.json>`

The normalized record retains the input SHA-256, not private raw content. A human must review it before Stage 2 commits anything.

## Evidence classifications

- `physical_in_room`: a human facilitator directly observed the named physical condition in the room.
- `remote_observed`: a human facilitator directly observed the session through a remote view; this is not physical in-room evidence.
- `participant_reported`: the participant voluntarily described an experience; it is not direct observation.
- `automated_ci` or `virtual_offscreen`: machine evidence only; it can never be promoted to physical/manual evidence.
- `not_tested` / `not_observed`: the condition was not actually performed or seen.

No class is certification. Leave every unperformed manual check exactly `not_tested`.
