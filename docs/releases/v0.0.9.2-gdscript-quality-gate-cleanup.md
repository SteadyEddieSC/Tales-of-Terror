## Behavior and authority position

This release intentionally changes no gameplay, authored rules, role/card/event/action identifiers, public method names or signatures, protocol fields, node paths, scenes, project settings, renderer, viewport, visual design, branding, assets, balance, native-authority boundaries, companion behavior, dependencies, export, or deployment behavior. Formatter-only files are separated from manual lint corrections in Git history, and a whitespace-insensitive semantic review is recorded in the committed evidence matrix.

Godot remains the sole gameplay authority. Role projection/evaluation helpers are deterministic and read-only; `RoleSession` retains all state mutation, transition, action, outcome commit, audit, and rejection ownership. Rules and board changes still cross their existing public validation boundaries, and invalid work consumes no gameplay RNG.

## Toolchain position

The release preserves official Godot 4.7.1-stable, the Compatibility renderer, the 960×540 logical viewport, GUT 9.7.1, Python 3.11.9, `gdtoolkit==4.5.0`, the complete exact/hash-locked Python graph, Node/npm dependencies, and immutable action commits.
