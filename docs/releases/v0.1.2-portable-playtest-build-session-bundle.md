# v0.1.2 — Portable Playtest Build & Session Bundle

## Outcome

This bounded release turns the accepted Lantern House vertical slice into internal Windows and Linux x86_64 workflow artifacts without committing binaries or widening gameplay scope. Official Godot 4.7.1 export templates are checksum-pinned. Exact bundle allowlists, denylist checks, file hashes, deterministic content identities, relative offline launchers, and an explicitly non-deterministic build timestamp make each artifact inspectable without overstating reproducibility.

The bundle carries a start guide, facilitator guide, post-session questionnaire, privacy/limitations notice, default manual hardware-validation record, third-party notices, and the Godot engine license. The native executable remains directly launchable; the helper is optional and requires neither administrator access nor a network connection.

## Support identity and authority

Help page 4 presents only release, short source commit, target, scenario/report schema, generic report location, reset instructions, and safe problem-reporting guidance. The generated identity is ignored source input created only for export. It never enters gameplay state, snapshots, RNG, reports, authority/public-history digests, or companion protocol. Source checkouts receive a bounded fallback identity.

The existing v0.1.1 title-to-rematch and no-phone routes, 1–8 stable-seat controller-first behavior, keyboard fallback, local schema-v2 report export, reset/rollback/rematch determinism, and optional non-authoritative companions remain unchanged.

## Honest execution scope

Automated exported-build smoke verifies that the real main scene loads, the build/support Help page is reachable, expected bounded identity/report guidance is visible, and coordinator authority remains unchanged. Windows execution is classified as local native/headless smoke on Windows; Linux execution is classified as CI native/headless smoke on Ubuntu. Neither is physical-controller, television-distance, household, physical-phone, assistive-technology, long-session, or network-environment validation.

All committed manual hardware checks remain `not_tested`. The evidence matrix records exact local/CI commands, outcomes, source heads, artifacts, hashes, warnings, corrections, and reruns.
