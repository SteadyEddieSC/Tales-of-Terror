# v0.1.3 — First Household/Remote Playtest Pilot & Findings Triage

Stage 1 prepares one frozen internal Windows/Linux candidate, a consent-based facilitator kit, exact blank pilot/findings schemas, and offline privacy validation. It changes no gameplay, balance, pacing, content, companion authority, architecture, dependency, toolchain, or final branding decision.

The committed records contain no human findings. Every human/manual value begins as `not_tested`, `not_observed`, false, zero, or empty. Offline intake accepts one exact record file, requires the frozen artifact identity, rejects ambiguous/private data, and produces deterministic normalized output with input-file SHA-256 provenance. Deterministic triage formats only human-entered judgments and cannot infer intent, emotion, consent, fun, accessibility compliance, or severity.

The draft release remains incomplete until a real, voluntary human session uses the exact frozen candidate identified by the `PILOT-READY HANDOFF`. A changed build requires a new candidate handoff. One pilot is not certification of balance, fun, duration, accessibility, privacy, security, or production readiness.
