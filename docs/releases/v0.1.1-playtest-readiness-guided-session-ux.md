# v0.1.1 — Playtest Readiness & Guided Session UX

## Scope

This release prepares the accepted Lantern House vertical slice for first meaningful household or remote-observed sessions. It adds presentation guidance, controller help/accessibility, interruption messaging, and explicit local playtest reports. It does not add or rebalance gameplay, content, authority, networking, deployment, dependencies, or final branding.

## Player-facing changes

- The complete title-to-rematch route now explains joining/leaving, mode fallback, objective, current stage, prompt/vote submitted counts, expected stable seats, pause, reconnect, reset, ending, rematch, and return.
- Controller A is ownership- and lifecycle-aware: the first press claims an unowned stable seat without double-advancing, a later press by an owner locks the roster, and additional controllers retain their join window before confirmation. Joypad bindings apply to every connected device; Enter follows keyboard ownership and Space remains a confirm fallback.
- X on a controller or H on a keyboard opens a four-page high-contrast help surface in every supported lifecycle state. Gameplay input is blocked while it is visible; authority state remains unchanged.
- Finalized sessions can be explicitly exported as exact-schema JSON and human-readable Markdown under the local Godot user-data `playtest_exports` folder.
- Schema v2 separates `ending`/`reset` completion from a one-time `pending`/`rematch`/`return_to_title`/`reset` post-ending disposition, preserving ending export and completed authority/public-history digests without silent relabeling or overwrite.

## Privacy and determinism

The reporter receives only allowlisted public state, reduced seat rows, and companion aggregates. It records no role/faction/objective secret, unresolved choice, room secret, token, client, device, machine, path, or network identity and has no transmission path. Observation-on and observation-off fixtures produce the same authority and public-history digests for seed 12031.

## Validation summary

The standalone readiness suite covers lifecycle guidance, help invariance, prompt/vote waits, disconnect/reconnect, protected resets, schema/bounds/order, planted secrets, JSON/Markdown consistency, export success/failure/path collision/path rejection, completion/disposition semantics, no-phone completion, companion non-authority, and no-network source proof. A separate main-scene suite drives real controller/key event objects through the complete non-active lifecycle and integrated report routes. Full-route captures compose `Main.tscn`, `ExplorationSandbox`, its CanvasLayer HUD, compact guidance, help, and ending. Focused GUT coverage exercises the model boundary in the vendored runner. Every inherited test and simulation remains in CI.

Physical controllers, phones, household Wi-Fi, television-distance readability, physical/native 4K, router/firewall variation, live Cloudflare, penetration certification, balance/fun, duration, and long-session observation remain deferred and unclaimed.
