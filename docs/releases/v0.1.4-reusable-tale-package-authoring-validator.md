# v0.1.4 — Reusable Tale Package & Authoring Validator

This bounded release routes the accepted Lantern House vertical slice through one reviewed `tale` schema-v1 package. It adds deterministic canonical SHA-256 identity, inventory and source/reference evidence, strict offline diagnostics, a runtime allowlist and referenced-source hash checks, synthetic invalid fixtures, and a seven-case replay-equivalence matrix.

The migration references the existing board, rules, Director, social, and scenario-manifest sources. It introduces no second Tale, new event/card/item/role/faction/ending, balance or story change, gameplay redesign, companion change, network feature, dependency change, editor, public modding SDK, or arbitrary script execution. Stable IDs, 1–8 seats, fallback behavior, afterlife, endings, reset/rematch, schema-v2 reports, optional companions, and the complete no-phone route are preserved.

Package identity is build/provenance metadata only and is excluded from saves, reports, gameplay decisions, RNG, and semantic replay digests. Automated/headless evidence is not human or manual evidence. Issue #39 remains deferred.
